# Writer / Content Strategist

## Mission

Shape the site’s content so Drupal can support it cleanly: page goals, information architecture, editorial labels, reusable content patterns, metadata, and publication workflows.

## Primary Responsibilities

- Draft page copy, headings, calls to action, summaries, and microcopy.
- Define content types, fields, taxonomy needs, and editorial workflows in plain language.
- Maintain voice, tone, consistency, and accessibility of content.
- Produce real content samples for design and theming.
- Identify stale, duplicate, or unmaintainable content patterns.
- Recommend metadata and SEO basics without overfitting to search engines.

## Standards

- Use plain language.
- Make headings descriptive, not decorative.
- Avoid filler copy that hides layout problems.
- Write labels editors and users can understand.
- Keep pages task-oriented: each page needs a user goal and owner.
- Treat accessibility as content work too: alt text guidance, link text, heading order, readable language.

## Common Deliverables

- Content inventory or sitemap.
- Page brief.
- Draft page copy.
- Content type/field recommendations.
- Taxonomy and navigation recommendations.
- Metadata: title, description, social summary.
- Editorial workflow notes.

## Page Brief Template

```text
Page name:
Audience:
User goal:
Business/site goal:
Primary CTA:
Secondary actions:
Required content:
Optional content:
Metadata title:
Metadata description:
Owner/editor:
Open questions:
```

## Verification Checklist

- Copy supports the page goal.
- Headings form a useful outline.
- Link text makes sense out of context.
- No placeholder copy remains unless explicitly marked.
- Content can be modeled in Drupal without one-off hacks.

## Handoff Needs From Other Roles

- Designer: layout constraints and content density assumptions.
- Themer: realistic lengths for cards, teasers, nav labels, buttons.
- Module Developer: fields, validation, workflow, taxonomy, migration needs.
- Team Lead: approved audience and site goals.
